How to rate the metrics

Severity
Low: visual or minor issue. Medium: common workflow is degraded. High: important feature is broken. Critical: outage, data loss, or security risk.
Impact: 1-5
1 = one user or rare case, 3 = frequent workflow, 5 = many users blocked or revenue-critical path.
Similar reports
0-2 is low, 3-9 means repeated signal, 10+ means the issue is widespread.
Module importance: 1-5
1 = optional area, 3 = regular product flow, 5 = core business functionality such as login, payment, or production reporting.
